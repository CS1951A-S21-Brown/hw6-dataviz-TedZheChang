2.

  • One reason why d3 was helpful was because it factors out alot of the scaling and size calculations. I didn't have to worry about manually finding the spacing of the bars or the length they had to be.
  • Another helpful part of d3 is that it supports easily transitioning between different set/subsets of data. As a result when trying to show the rankings between different years or genres it was smooth to create/transition between these different graphs.
  • Lastly d3 was helpful for my visualization because it allows one to attach event listeners and annotations to anything that is drawn. As such it was easier to annotate and add tooltips that help to give more information in a readable way for the data visualsations.

3.

  • One reason someone might not want to use d3 is that it does not have any built in statistcial model or calculations. As a result, unlike other graphing software which includes these features, one would have to separately perform these analyses then manually figure out how to draw them onto the graph, which can be difficult.
  • Another reason to not use d3 is that it relies on the browser/page being allowed to use javascript. As such if javascript is not enabled then the visualization would not work. On the other hand something like an image or uploaded svg inserted into the DOM would always load.
  • D3 would also not be the best tool for creating a visualization if one is not very familiar with javascript/is using a pipeline that is hard to connect to javascript. In situations like this it would be easier to use a graphing software that one is more comfortable with/ is more well suited to the pipeline one is using.

4. One group that would find my dashboard fairly accessible are people use assitive technologies such as screen readers. This is because I avoid using tools like overlays or sliders when displaying data or asking for use input, which is helpful for assistive technology according to the guide for accesible visualizations under the assistive technology section. A group that may find my dashboard more difficult to view though are certain groups with colorblindness. This is because I don't have much color contrast on my graphs, with everything being a shade of blue or purple. As such, according to the color blindess section of the same guide, some parts of my graph may be more difficult to distinguish. As such some actions that I could take would be to choose more contrasting colors for my graphs to aid in colorblindness. Additionally I could run my dashboard through some colorblindess filters to better verify which parts of my dashboard need improvements for contrasts, shapes, or anything else.
